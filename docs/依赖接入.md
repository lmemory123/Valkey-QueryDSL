# 依赖接入

这篇文档专门回答接入阶段最容易卡住的几件事：

- Maven 依赖怎么写
- Gradle 依赖怎么写
- APT / KAPT 怎么配置
- 稳定版、候选版、源码快照分别该用哪个坐标

## 1. 版本与坐标

当前推荐版本：

- 稳定版：`io.github.lmemory123:*:1.0.0`
- 候选版：`io.github.lmemory123:*:1.1.0-RC3`
- 当前源码快照：`com.momao:*:1.1.0-SNAPSHOT`

第一次接入时，优先级建议是：

1. 生产或正式项目，优先用稳定版
2. 想提前验证新能力，用 `RC`
3. 只有在你明确需要最新未发布代码时，才用源码快照

## 2. Maven 依赖

### 2.1 业务依赖

```xml
<dependency>
    <groupId>io.github.lmemory123</groupId>
    <artifactId>valkey-query-spring-boot-starter</artifactId>
    <version>1.1.0-RC3</version>
</dependency>
```

### 2.2 APT 处理器

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <annotationProcessorPaths>
            <path>
                <groupId>io.github.lmemory123</groupId>
                <artifactId>valkey-query-processor</artifactId>
                <version>1.1.0-RC3</version>
            </path>
        </annotationProcessorPaths>
    </configuration>
</plugin>
```

### 2.3 源码快照写法

如果你要接入当前工作区的最新代码，先在本仓库执行：

```bash
mvn clean install
```

然后在业务项目里引用：

```xml
<dependency>
    <groupId>com.momao</groupId>
    <artifactId>valkey-query-spring-boot-starter</artifactId>
    <version>1.1.0-SNAPSHOT</version>
</dependency>
```

APT 处理器也要跟着切到同一组坐标：

```xml
<path>
    <groupId>com.momao</groupId>
    <artifactId>valkey-query-processor</artifactId>
    <version>1.1.0-SNAPSHOT</version>
</path>
```

## 3. Gradle 依赖

### 3.1 Java 项目

```kotlin
dependencies {
    implementation("io.github.lmemory123:valkey-query-spring-boot-starter:1.1.0-RC3")
    annotationProcessor("io.github.lmemory123:valkey-query-processor:1.1.0-RC3")
}
```

### 3.2 源码快照

```kotlin
dependencies {
    implementation("com.momao:valkey-query-spring-boot-starter:1.1.0-SNAPSHOT")
    annotationProcessor("com.momao:valkey-query-processor:1.1.0-SNAPSHOT")
}
```

## 4. Kotlin / KAPT

如果是 Kotlin 项目，处理器依赖改成 `kapt`：

```kotlin
plugins {
    kotlin("kapt")
}

dependencies {
    implementation("io.github.lmemory123:valkey-query-spring-boot-starter:1.1.0-RC3")
    kapt("io.github.lmemory123:valkey-query-processor:1.1.0-RC3")
}
```

如果你要接源码快照，把坐标切到 `com.momao:*:1.1.0-SNAPSHOT` 即可。

## 5. 接入时最容易卡住的坑

### 5.1 Starter 和 Processor 版本不一致

例如：

- Starter 用了 `1.1.0-RC3`
- Processor 还写着 `1.0.0`

这种很容易导致：

- APT 生成结果不一致
- 新能力在编译期不存在
- 运行时和编译期元数据错位

最稳的原则：

- `starter` 和 `processor` 始终用同一个版本号

### 5.2 文档坐标和源码坐标混用

这个项目发布态和开发态坐标不同：

- 发布态：`io.github.lmemory123`
- 开发态：`com.momao`

不要把这两套混着抄。

### 5.3 只加 Starter，不配 APT

如果没配 APT，就不会生成：

- `SkuQuery`
- `StudentQuery`
- `XXXQuery.METADATA`

这不是运行时问题，是编译阶段没接好。
