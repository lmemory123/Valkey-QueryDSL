# 真实环境测试手册

更新时间：2026-04-04

本文档用于统一 `Valkey QueryDSL` 的测试原则与本地验证方式。

## 一、测试原则

项目当前采用“真实 Valkey 环境优先”的测试策略。

约束如下：

- 正确性结论以真实 Valkey 集成测试为准
- 不使用 Mockito 作为运行时行为正确性的主要依据
- standalone、read_write_split、cluster 三种拓扑都必须有真实验证
- 纯模型 / 纯 DSL / 纯编译期测试只用于补充静态回归，不作为运行时正确性结论

## 二、本地环境约定

当前默认本地端口如下：

- standalone：`localhost:6379`
- read_write_split：
  - primary：`localhost:6381`
  - replica：`localhost:6382`
- cluster：
  - seed：`localhost:8000`
  - node2：`localhost:8001`
  - node3：`localhost:8002`

## 五、已确认的真实环境差异

以下问题都不是 mock 测试能稳定发现的：

- 副本节点可能没有 Search 索引元数据，`FT.*` 读请求需要回退主库
- cluster 更适合使用单 seed 真实接入，不应在测试里伪造多 seed 透明接入
- 默认索引治理模式为 `NONE` 时，集成测试必须显式建索引
- macOS Docker Desktop 下，本地 3 节点 cluster 推荐使用 `host.docker.internal` 作为 `cluster-announce-ip`，否则 cluster bus 容易因为 `127.0.0.1` 回环导致 `cluster_state:fail`
- `COUNT_DISTINCT` 当前真实服务端只按“带 `GROUPBY` 的 reducer”对外承诺，cluster 下不支持
- `countDistinct(field)` 当前只对单字段精确计数对外承诺，底层会顺序分页遍历 distinct 结果
- `facetChain.minCount(...)` 当前按客户端安全过滤实现，对 standalone / read_write_split / cluster 三种拓扑都已有真实验证
- `facetChain.includeValues(...)` 当前按客户端安全过滤实现，对 standalone / read_write_split / cluster 三种拓扑都已有真实验证
- 向量 KNN 当前已在 standalone / read_write_split 下真实跑通，在 cluster 下已真实验证为当前不支持

## 六、后续规则

后续新增功能时，至少满足以下要求：

1. 如果改动影响 Repository 运行时行为，必须补真实 Valkey 集成测试
2. 如果改动影响拓扑路由，必须至少覆盖 read_write_split 或 cluster 的真实场景
3. 如果改动只影响编译期代码生成或纯模型构造，可以补充纯测试，但不能替代真实集成测试
