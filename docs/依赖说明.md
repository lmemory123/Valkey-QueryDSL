# 依赖说明

这篇文档说明 `Valkey QueryDSL` 当前依赖了什么，以及这些依赖分别承担什么职责。

## 1. 运行环境

- Java `17`
- Maven `3.9+`
- Spring Boot `4.0.3`

根 `pom.xml` 中统一管理版本：

- `spring-boot.version = 4.0.3`
- `valkey-glide.version = 2.2.7`
- `javapoet.version = 1.13.0`
- `auto-service.version = 1.1.1`

## 2. 核心三方依赖

### 2.1 `io.valkey:valkey-glide`

用途：

- 作为底层 Valkey 客户端
- 承担真实命令执行
- 提供 `FT.create`、`FT.search`、`GlideClient` 等 API

当前项目中的作用：

- Starter 负责创建 `GlideClient`
- Adapter 层负责把查询条件转换成真正的 Valkey 请求

说明：

- 当前通过 Maven Profile 自动切换本地平台 classifier
- 根 `pom.xml` 已为 macOS / Linux / Windows 配置了 classifier 选择

### 2.2 `com.fasterxml.jackson.core:jackson-databind`

用途：

- JSON 序列化 / 反序列化
- HASH 场景下字段名映射
- 实体结果映射

当前项目中的作用：

- `JSON` 文档读写
- `HASH` 结构对象转换
- `SNAKE_CASE` 字段映射

### 2.3 `org.springframework.boot:spring-boot-starter`

用途：

- 提供自动配置、生命周期、配置绑定和 Bean 管理能力

当前项目中的作用：

- `valkey-query-spring-boot-starter` 的运行基础
- `@EnableValkeyQuery`、配置属性、启动后索引创建

## 3. 编译期依赖

### 3.1 `com.squareup:javapoet`

用途：

- 生成 Java 源码

当前项目中的作用：

- `valkey-query-processor` 在编译期生成 `XXXQuery`
- 同时生成 `METADATA`

### 3.2 `com.google.auto.service:auto-service-annotations`

用途：

- 标记注解处理器

当前项目中的作用：

- 让 `ValkeyQueryProcessor` 被编译器识别为标准 APT Processor

### 3.3 `com.google.auto.service:auto-service`

用途：

- 生成 Processor 注册文件

当前项目中的作用：

- 通过 `maven-compiler-plugin` 的 `annotationProcessorPaths` 参与编译

### 3.4 `org.springframework.boot:spring-boot-configuration-processor`

用途：

- 生成配置元数据

当前项目中的作用：

- 为 `ValkeyQueryProperties` 生成 Spring Boot 配置提示

## 4. 模块间依赖关系

### `valkey-query-annotations`

依赖：

- 无额外三方依赖

职责：

- 定义注解和枚举

### `valkey-query-core`

依赖：

- `valkey-query-annotations`

职责：

- 查询模型
- Repository 接口
- 分页对象
- 元数据对象

### `valkey-query-processor`

依赖：

- `valkey-query-annotations`
- `valkey-query-core`
- `javapoet`
- `auto-service-annotations`

职责：

- 编译期生成 `XXXQuery`

### `valkey-query-glide-adapter`

依赖：

- `valkey-query-core`
- `jackson-databind`
- `valkey-glide`

职责：

- Repository 基类
- 命令执行
- 结果映射
- JSON / HASH 存储策略

### `valkey-query-spring-boot-starter`

依赖：

- `valkey-query-annotations`
- `valkey-query-core`
- `valkey-query-glide-adapter`
- `spring-boot-starter`
- `spring-boot-configuration-processor`

职责：

- 自动配置
- 连接属性绑定
- 包扫描
- 索引自动创建

### `valkey-query-test-example`

依赖：

- `valkey-query-spring-boot-starter`
- `spring-boot-starter-web`
- `spring-boot-starter-test`

职责：

- 演示接入方式
- 承载集成测试和回归验证

## 5. 兼容性说明

### Java 版本

当前代码和 APT 均按 `Java 17` 编译。

### Spring Boot

根 `pom.xml` 当前管理的是 `Spring Boot 4.0.3`。

### Valkey Glide

当前使用 `2.2.7`。

已知实现特征：

- 无排序查询走 Glide 的 `FT.search`
- 有排序查询走真实 `FT.SEARCH ... SORTBY ... LIMIT ...`
- `count()` 走 `FT.SEARCH ... LIMIT 0 0`

## 6. 对外暴露的核心依赖面

如果你是使用方，通常只需要关心两件事：

1. 运行时依赖 `valkey-query-spring-boot-starter`
2. 编译时在 `maven-compiler-plugin` 中加入 `valkey-query-processor`

其余依赖由项目内部模块统一收口。
