# 快速开始

这篇文档的目标是让你在一个新的 Spring Boot 项目中，把 `Valkey QueryDSL` 接入并跑通第一条查询。

## 1. 前置条件

- Java 17+
- Maven 3.9+
- 可用的 Valkey 服务
- 服务端具备 Search / JSON 能力

## 2. 选择接入方式

推荐先区分两种场景：

- 已发布稳定版：直接使用 Maven Central 上的 `io.github.lmemory123:1.0.0`
- 已发布候选版：直接使用 Maven Central 上的 `io.github.lmemory123:1.1.0-RC5`
- 当前源码快照：在本仓库本地执行 `mvn clean install` 后，使用 `com.momao:1.1.0-SNAPSHOT`

### 2.1 使用已发布稳定版

业务项目至少需要引入 Starter：

```xml
<dependency>
    <groupId>io.github.lmemory123</groupId>
    <artifactId>valkey-query-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

并为编译器配置 APT：

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <annotationProcessorPaths>
            <path>
                <groupId>io.github.lmemory123</groupId>
                <artifactId>valkey-query-processor</artifactId>
                <version>1.0.0</version>
            </path>
        </annotationProcessorPaths>
    </configuration>
</plugin>
```

### 2.2 使用当前源码快照

如果你要在其他项目中引用当前工作区里的最新源码，先在本仓库根目录执行：

```bash
mvn clean install
```

当前源码快照会安装到本机 Maven 仓库：

- `com.momao:valkey-query-annotations:1.1.0-SNAPSHOT`
- `com.momao:valkey-query-core:1.1.0-SNAPSHOT`
- `com.momao:valkey-query-processor:1.1.0-SNAPSHOT`
- `com.momao:valkey-query-glide-adapter:1.1.0-SNAPSHOT`
- `com.momao:valkey-query-spring-boot-starter:1.1.0-SNAPSHOT`

对应依赖写法：

```xml
<dependency>
    <groupId>com.momao</groupId>
    <artifactId>valkey-query-spring-boot-starter</artifactId>
    <version>1.1.0-SNAPSHOT</version>
</dependency>
```

APT 配置：

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <annotationProcessorPaths>
            <path>
                <groupId>com.momao</groupId>
                <artifactId>valkey-query-processor</artifactId>
                <version>1.1.0-SNAPSHOT</version>
            </path>
        </annotationProcessorPaths>
    </configuration>
</plugin>
```

## 3. 启用扫描

```java
@SpringBootApplication
@EnableValkeyQuery(basePackages = "com.example.demo")
public class DemoApplication {
}
```

从 `1.1.0-RC5` 开始，下面两种写法也可以直接用：

```java
@EnableValkeyQuery("com.example.demo")
```

```java
@EnableValkeyQuery
```

说明：

- `value` 是 `basePackages` 的别名
- 如果你不传包名，默认会回退到启动类所在包
- 单模块 Spring Boot 项目优先用无参写法，接入负担最低

如果你不想自己拼完整示例，直接看：

## 4. 配置连接

第一次接入先用最小配置：

```yaml
valkey:
  query:
    enabled: true
    mode: standalone
    nodes:
      - host: 127.0.0.1
        port: 6379
```

如果服务端开了密码，再补：

```yaml
valkey:
  query:
    username: default
    password: your-password
```

## 5. 定义实体

```java
@ValkeyDocument(
    indexName = "idx:sku",
    prefixes = {"sku:"},
    storageType = StorageType.JSON
)
public class Sku {

    @ValkeyId
    private String id;

    @ValkeySearchable(weight = 2.5d, noStem = true)
    private String title;

    @ValkeyNumeric(sortable = true)
    private Integer price;

    @ValkeyTag
    private List<String> tags;

    @ValkeyIndexed
    private Merchant merchant;
}
```

```java
public class Merchant {
    private String name;
    private String level;
}
```

编译后会生成 `SkuQuery`。

## 6. 定义 Repository

```java
@Repository
public class SkuRepository extends BaseValkeyRepository<Sku> {

    public SkuRepository(
            ValkeyClientRouting clientRouting,
            ObjectMapper objectMapper) {
        super(SkuQuery.METADATA, clientRouting, Sku.class, objectMapper);
    }
}
```

## 7. 发起查询

```java
SkuQuery q = new SkuQuery();

Page<Sku> page = skuRepository.queryChain()
    .where(q.title.contains("iphone"))
    .and(q.merchant.name.eq("Apple Flagship"))
    .orderByAsc("price")
    .page(0, 20);
```

或者直接使用 Repository：

```java
SearchCondition condition = q.title.contains("iphone").sortBy("price", true);

List<Sku> list = skuRepository.list(condition);
Page<Sku> result = skuRepository.page(condition, 0, 20);
Sku one = skuRepository.one(q.id.eq("sku-1"));
long total = skuRepository.count(q.tags.eq("HOT"));
```
