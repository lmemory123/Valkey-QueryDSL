# 基础功能

这篇文档只讲最常用的主链能力：

- 增
- 删
- 改
- 基础查询
- 批量操作
- 链式操作

## 1. 保存

```java
skuRepository.save("sku-1", sku);
```

如果实体本身带 `id` 字段，也可以直接让业务对象承担主键内容，再按业务层约定传入 key。

## 2. 基础查询

### 2.1 `list`

```java
SkuQuery q = new SkuQuery();

List<Sku> records = skuRepository.list(
    q.price.between(1000, 5000)
        .and(q.tags.eq("HOT"))
);
```

### 2.2 `page`

```java
Page<Sku> page = skuRepository.page(
    q.title.contains("iphone"),
    0,
    20
);
```

### 2.3 `one`

```java
Sku one = skuRepository.one(q.id.eq("sku-1"));
```

### 2.4 `count`

```java
long total = skuRepository.count(q.status.eq("ON"));
```

## 3. 链式查询

`queryChain()` 是最常用的主入口。

```java
Page<Sku> page = skuRepository.queryChain()
    .where(q.tags.eq("HOT"))
    .and(q.merchant.name.eq("Apple"))
    .orderByAsc("price")
    .page(0, 10);
```

常用能力：

- `where(...)`
- `and(...)`
- `or(...)`
- `orderByAsc(...)`
- `orderByDesc(...)`
- `list()`
- `page(...)`
- `one()`
- `count()`

## 4. 局部更新

`updateChain()` 用于单 key 的条件更新。

```java
long updated = skuRepository.updateChain()
    .where(q.id.eq("sku-1"))
    .and(q.price.eq(1999))
    .set(q.price, 2999)
    .execute();
```

当前语义：

- 更新成功返回 `1`
- key 不存在返回 `0`
- 条件不匹配也返回 `0`

### 4.1 乐观锁

```java
long updated = skuRepository.updateChain()
    .where(q.id.eq("sku-1"))
    .expectVersion(q.version, 3)
    .set(q.title, "new-title")
    .advanceVersion(q.version, 3)
    .execute();
```

### 4.2 原子增减

```java
studentRepository.updateChain()
    .where(q.id.eq("1"))
    .increment(q.age, 1)
    .execute();
```

说明：

- 只接受数值字段
- 单字段无条件增减优先走原生命令
- 混合条件更新会回退到单 key Lua 原子脚本

## 5. 批量操作

### 5.1 批量保存

```java
BulkWriteResult result = skuRepository.saveAll(items, BulkWriteOptions.unordered());
```

### 5.2 批量删除

```java
BulkWriteResult result = skuRepository.deleteAll(ids, BulkWriteOptions.ordered());
```

### 5.3 批量更新

```java
BulkWriteResult result = skuRepository.updateAll(items, BulkWriteOptions.unordered());
```

当前批量写语义：

- 支持 `ORDERED / UNORDERED`
- 返回统一 `BulkWriteResult`
- cluster 下会按节点分组执行

## 6. 自动映射

当前框架默认做两类映射：

1. 查询结果到实体对象
2. 文档 key 回填到 `id`

已验证的结构：

- 扁平字段
- `List<String>`
- JSON 嵌套对象

## 7. 推荐使用方式

日常业务建议优先这样分层：

- 普通查询：`list / page / one / count`
- 复杂过滤：`queryChain()`
- 局部写入：`updateChain()`
- 多条写操作：`saveAll / deleteAll / updateAll`
