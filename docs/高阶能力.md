# 高阶能力

本文档聚焦 5 类能力：

- Projection
- 聚合
- Facet
- Distinct
- 向量检索

先看边界：

- standalone：全部按文档能力支持
- `read_write_split`：全部按文档能力支持
- cluster：只支持明确承诺的安全子集

## 1. Projection

### 1.1 实体裁剪查询

```java
List<Sku> records = skuRepository.queryChain()
    .where(q.tags.eq("HOT"))
    .select("title", "price", "merchant")
    .list();
```

说明：

- `HASH` 会生成真实 `RETURN field...`
- `JSON` 会生成真实 `RETURN $.path AS alias ...`
- 返回结果仍映射到实体，只是未选字段为空

### 1.2 零反射 DTO 投影

```java
List<SkuSummary> rows = skuRepository.queryChain()
    .where(q.tags.eq("HOT"))
    .list(sku -> new SkuSummary(sku.getTitle(), sku.getPrice()));
```

这是当前推荐方式，因为它是零反射。

## 2. 聚合

### 2.1 基础聚合

```java
AggregateResult result = repository.aggregateChain()
    .where(q.status.eq("ON"))
    .groupBy(q.department)
    .count("count")
    .avg(q.score, "avg_score")
    .sortByDesc("count")
    .limit(0, 10)
    .result();
```

当前已支持：

- `GROUPBY`
- `COUNT`
- `COUNT_DISTINCT`
- `SUM`
- `AVG`
- `MIN`
- `MAX`
- `SORTBY`
- `LIMIT`

### 2.2 类型化结果映射

```java
record DepartmentStats(String department, long count, double avg_score) {}

List<DepartmentStats> rows = result.mapRows(row ->
    new DepartmentStats(
        row.getString("department"),
        row.getLong("count"),
        row.getDouble("avg_score")
    )
);
```

推荐入口：`mapRows(Function<AggregateRow, R>)`

这是零反射入口。`mapRows(Class)` 只是便捷模式。

### 2.3 APPLY / FILTER

```java
AggregateResult result = repository.aggregateChain()
    .groupBy(q.department)
    .count("count")
    .apply("@count * 10", "weighted_count")
    .filter("@weighted_count > 10")
    .result();
```

也支持类型化表达式 DSL。

## 3. Facet

### 3.1 单字段 Facet

```java
FacetResult facet = repository.facetChain()
    .where(q.status.eq("ON"))
    .on(q.department)
    .size(10)
    .sortByCountDesc()
    .result();
```

### 3.2 多字段独立 Facet

```java
FacetResults facets = repository.facetChain()
    .where(q.status.eq("ON"))
    .on(q.department, q.city)
    .size(10)
    .results();
```

### 3.3 过滤能力

```java
FacetResult facet = repository.facetChain()
    .on(q.department)
    .minCount(2)
    .includeValues("Backend", "Search")
    .result();
```

## 4. Distinct

### 4.1 单字段去重

```java
List<String> departments = repository.queryChain()
    .where(q.status.eq("ON"))
    .distinct(q.department);
```

### 4.2 去重结果映射

```java
List<AggregateRow> rows = repository.queryChain()
    .where(q.status.eq("ON"))
    .distinctRows(q.department, q.city);
```

### 4.3 去重计数

```java
long total = repository.queryChain()
    .where(q.status.eq("ON"))
    .countDistinct(q.department);
```

## 5. 向量检索

```java
List<VectorItem> records = vectorRepository.queryChain()
    .where(q.status.eq("ON"))
    .knn(q.embedding, new float[]{0.1f, 0.2f, 0.3f, 0.4f}, 10)
    .list();
```

当前边界：

- standalone：已真实验证
- `read_write_split`：已真实验证
- cluster：当前明确不支持

## 6. cluster 下的能力边界

cluster 不是“所有能力都透明支持”，而是明确收口成安全子集。

当前 cluster 已支持：

- 基础查询
- 批量写
- Projection 主链
- `GROUPBY + COUNT/SUM/AVG/MIN/MAX + SORTBY + LIMIT`
- 单字段和多字段独立 Facet
- `distinct / distinctRows / countDistinct`

当前 cluster 仍明确不支持：

- 向量 KNN
- 不可安全合并的聚合表达式组合

## 7. 推荐使用原则

- 运行时热路径优先使用零反射 API
- 高阶查询以真实 Valkey 验证结果为准
- cluster 下只使用文档明确承诺的能力子集
