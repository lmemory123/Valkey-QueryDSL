# 注解参考

这篇文档专门讲注解，不掺别的内容。

## `@ValkeyDocument`

`@ValkeyDocument` 用来描述：

- 这个实体对应哪个索引
- 用什么前缀生成 key
- 存储类型是 `JSON` 还是 `HASH`

定义示例：

```java
@ValkeyDocument(
    indexName = "idx:sku",
    prefixes = {"sku:"},
    storageType = StorageType.JSON
)
public class Sku {
}
```

常用属性：

```java
public @interface ValkeyDocument {
    String indexName();
    String[] prefixes() default {};
    StorageType storageType() default StorageType.JSON;
    String querySuffix() default "Query";
}
```

使用建议：

- 有集合字段、嵌套对象时优先 `JSON`
- 扁平结构、简单字段时可选 `HASH`

## `@ValkeyId`

`@ValkeyId` 标记主键字段。

```java
@ValkeyId
private String id;
```

它的作用：

- 标记业务主键字段
- 查询结果回填时，用文档 key 自动写回 `id`

当前行为说明：

- `id` 会被索引成 `TAG`
- 当前不会再为 `TAG` 字段生成 `SORTABLE`

## `@ValkeySearchable`

`@ValkeySearchable` 用于全文检索字段。

```java
@ValkeySearchable(weight = 2.5d, noStem = true)
private String title;
```

近似定义：

```java
public @interface ValkeySearchable {
    String value() default "";
    double weight() default 1.0d;
    boolean noStem() default false;
    boolean sortable() default false;
}
```

说明：

- `value` 用于自定义索引字段名
- `weight` 控制全文检索权重
- `noStem` 控制是否禁用词干提取
- `sortable` 当前只对 `TEXT` 字段有效

示例：

```java
@ValkeySearchable(value = "class_name", weight = 1.5d)
private String className;
```

## `@ValkeyIndexed`

`@ValkeyIndexed` 用于结构化索引字段。

```java
@ValkeyIndexed(sortable = true)
private Integer price;
```

近似定义：

```java
public @interface ValkeyIndexed {
    String value() default "";
    boolean sortable() default false;
}
```

字段类型推导规则：

- `String` -> `TAG`
- 数值类型 -> `NUMERIC`
- `List<String>` -> `TAG`
- 嵌套对象中的叶子字段 -> 按叶子字段类型推导

当前说明：

- `sortable=true` 对数值字段有效
- 如果最终推导成 `TAG`，当前会忽略 `sortable=true`

示例：

```java
@ValkeyIndexed("producer_mark")
private String producerMark;

@ValkeyIndexed(value = "play_count", sortable = true)
private Integer playCount;
```

## `@ValkeyVector`

`@ValkeyVector` 用于向量字段。

```java
@ValkeyVector(dimension = 4)
private float[] embedding;
```

当前边界：

- 当前只支持 `JSON`
- 当前支持 `float[]` / `double[]`
- standalone / `read_write_split` 已真实验证
- cluster 下当前明确不支持向量 KNN

## 注解组合示例

```java
@ValkeyDocument(
    indexName = "idx:music",
    prefixes = {"music:"},
    storageType = StorageType.JSON
)
public class MusicDocument {

    @ValkeyId
    private String id;

    @ValkeySearchable(weight = 2.0d, noStem = true)
    private String title;

    @ValkeyIndexed
    private String artist;

    @ValkeyIndexed(sortable = true)
    private Integer playCount;
}
```

## 注解使用时最容易踩的坑

### 1. `String` 字段默认不是 `TEXT`

如果你用的是：

```java
@ValkeyIndexed
private String title;
```

它默认会被当成 `TAG`，不是全文字段。  
要做全文搜索，应该写：

```java
@ValkeySearchable
private String title;
```

### 2. `TAG` 字段不要期待 `sortable=true`

当前框架已经明确收口：

- `TAG` 字段不生成 `SORTABLE`

原因是当前真实服务端下，这种组合不稳定，也会直接导致建索引失败。

### 3. 嵌套对象不是直接整对象索引

例如：

```java
private Merchant merchant;
```

实际索引的是叶子字段，比如：

- `merchant.name`
- `merchant.level`

而不是整个对象 blob。
