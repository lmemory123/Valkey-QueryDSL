# 注解参考

## `@ValkeyDocument`

`@ValkeyDocument` 用来描述：

- 这个实体对应哪个索引
- 用什么前缀生成 key
- 存储类型是 `JSON` 还是 `HASH`

定义示例：

```java
@ValkeyDocument(
    indexName = "idx:sku",
    prefixes = {"sku:"},
    storageType = StorageType.JSON
)
public class Sku {
}
```

常用属性：

```java
public @interface ValkeyDocument {
    String indexName();
    String[] prefixes() default {};
    StorageType storageType() default StorageType.JSON;
    String querySuffix() default "Query";
}
```

使用建议：

- 有集合字段、嵌套对象时优先 `JSON`
- 扁平结构、简单字段时可选 `HASH`

## `@ValkeyId`

`@ValkeyId` 标记主键字段。

```java
@ValkeyId
private String id;
```

它的作用：

- 标记业务主键字段
- 查询结果回填时，用文档 key 自动写回 `id`

当前行为说明：

- `id` 会被索引成 `TAG`
- 当前不会再为 `TAG` 字段生成 `SORTABLE`

## `@ValkeySearchable`

`@ValkeySearchable` 用于全文检索字段。

```java
@ValkeySearchable(weight = 2.5d, noStem = true)
private String title;
```

近似定义：

```java
public @interface ValkeySearchable {
    String value() default "";
    double weight() default 1.0d;
    boolean noStem() default false;
    boolean sortable() default false;
}
```

说明：

- `value` 用于自定义索引字段名
- `weight` 控制全文检索权重
- `noStem` 控制是否禁用词干提取
- `sortable` 当前只对 `TEXT` 字段有效

示例：

```java
@ValkeySearchable(value = "class_name", weight = 1.5d)
private String className;
```

## `@ValkeyTag`

`@ValkeyTag` 用于显式声明 `TAG` 字段。

```java
@ValkeyTag("status_tag")
private String status;
```

近似定义：

```java
public @interface ValkeyTag {
    String value() default "";
}
```

说明：

- 只表达一件事：这个字段就是 `TAG`
- 不再混入 `TEXT / NUMERIC` 语义
- `TAG` 当前不会生成 `SORTABLE`

示例：

```java
@ValkeyTag("producer_mark")
private String producerMark;

@ValkeyTag("status_tag")
private String status;
```

## `@ValkeyNumeric`

`@ValkeyNumeric` 用于显式声明 `NUMERIC` 字段。

```java
@ValkeyNumeric(value = "play_count", sortable = true)
private Integer playCount;
```

近似定义：

```java
public @interface ValkeyNumeric {
    String value() default "";
    boolean sortable() default false;
}
```

说明：

- 只表达一件事：这个字段就是 `NUMERIC`
- `sortable=true` 在这里是有效的
- 非数值 Java 字段使用 `@ValkeyNumeric` 会在编译期报错

## `@ValkeyIndexed`

`@ValkeyIndexed` 现在退回成“兼容旧写法 + 自动推断”的结构化注解。

```java
@ValkeyIndexed(sortable = true)
private Integer price;
```

当前自动推断规则：

- 数值类型 -> `NUMERIC`
- 其他结构化字段 -> `TAG`
- JSON 嵌套对象 -> 递归展开子字段

推荐策略：

- 新代码优先用 `@ValkeyTag / @ValkeyNumeric`
- 嵌套对象字段当前继续使用 `@ValkeyIndexed`
- 旧代码保留 `@ValkeyIndexed` 也还能工作

如果你要全文字段，请直接用：

```java
@ValkeySearchable(value = "keyword_text", sortable = true)
private String keyword;
```

## `@ValkeyVector`

`@ValkeyVector` 用于向量字段。

```java
@ValkeyVector(dimension = 4)
private float[] embedding;
```

当前边界：

- 当前只支持 `JSON`
- 当前支持 `float[]` / `double[]`
- standalone / `read_write_split` 已真实验证
- cluster 下当前明确不支持向量 KNN

## 注解组合示例

```java
@ValkeyDocument(
    indexName = "idx:music",
    prefixes = {"music:"},
    storageType = StorageType.JSON
)
public class MusicDocument {

    @ValkeyId
    private String id;

    @ValkeySearchable(weight = 2.0d, noStem = true)
    private String title;

    @ValkeyTag
    private String artist;

    @ValkeyNumeric(sortable = true)
    private Integer playCount;
}
```

## 注解使用时最容易踩的坑

### 1. `String` 字段默认不是 `TEXT`

如果你用的是：

```java
@ValkeyIndexed
private String title;
```

它默认会被当成 `TAG`，不是全文字段。  
要做全文搜索，应该写：

```java
@ValkeySearchable
private String title;
```

### 2. `TAG` 字段不要期待 `sortable=true`

当前框架已经明确收口：

- `TAG` 字段不生成 `SORTABLE`

原因是当前真实服务端下，这种组合不稳定，也会直接导致建索引失败。

### 3. `TEXT` 字段当前不会默认附带 `WITHSUFFIXTRIE`

当前框架不会再为普通 `TEXT` 字段无脑生成 `WITHSUFFIXTRIE`。

原因很简单：

- 注解层没有对应开关
- 真实服务端下，无条件拼接会把后续字段参数吃坏，直接导致 `FT.CREATE` 失败

如果后续要支持 suffix trie，必须先补明确注解或配置开关，而不是偷偷默认开启。

### 4. 嵌套对象不是直接整对象索引

例如：

```java
private Merchant merchant;
```

实际索引的是叶子字段，比如：

- `merchant.name`
- `merchant.level`

而不是整个对象 blob。
