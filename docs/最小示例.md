# 最小示例

这篇文档只做一件事：

- 给你一套可以直接复制进新项目的最小示例

目标不是覆盖全部能力，而是让你先把：

1. 依赖
2. 实体
3. Repository
4. 配置
5. 第一条查询

一次性跑通。

## 1. 完整 `pom.xml`

下面这份可以直接复制成一个最小 Maven 项目：

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>4.0.3</version>
        <relativePath/>
    </parent>

    <groupId>com.example</groupId>
    <artifactId>valkey-query-demo</artifactId>
    <version>0.0.1-SNAPSHOT</version>

    <properties>
        <java.version>17</java.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter</artifactId>
        </dependency>
        <dependency>
            <groupId>io.github.lmemory123</groupId>
            <artifactId>valkey-query-spring-boot-starter</artifactId>
            <version>1.1.0-RC5</version>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <configuration>
                    <annotationProcessorPaths>
                        <path>
                            <groupId>io.github.lmemory123</groupId>
                            <artifactId>valkey-query-processor</artifactId>
                            <version>1.1.0-RC5</version>
                        </path>
                    </annotationProcessorPaths>
                </configuration>
            </plugin>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
```

## 2. 启动类

```java
@SpringBootApplication
@EnableValkeyQuery
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

说明：

- 单模块项目直接用无参 `@EnableValkeyQuery`
- 默认会扫描启动类所在包

## 3. 实体

```java
@ValkeyDocument(
    indexName = "idx:book",
    prefixes = {"book:"},
    storageType = StorageType.JSON
)
public class Book {

    @ValkeyId
    private String id;

    @ValkeySearchable(weight = 2.0d)
    private String title;

    @ValkeyTag
    private String author;

    @ValkeyNumeric(sortable = true)
    private Integer price;

    public String getId() {
        return id;
    }

    public void setId(String id) {
        this.id = id;
    }

    public String getTitle() {
        return title;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public String getAuthor() {
        return author;
    }

    public void setAuthor(String author) {
        this.author = author;
    }

    public Integer getPrice() {
        return price;
    }

    public void setPrice(Integer price) {
        this.price = price;
    }
}
```

## 4. Repository

```java
@Repository
public class BookRepository extends BaseValkeyRepository<Book> {

    public BookRepository(
            ValkeyClientRouting clientRouting,
            ObjectMapper objectMapper) {
        super(BookQuery.METADATA, clientRouting, Book.class, objectMapper);
    }
}
```

说明：

- 当前最直接的方式仍然是继承 `BaseValkeyRepository`
- 最小示例统一用 2 参数版，首次接入心智负担最低
- 业务代码真正常用的是 `queryChain()` / `save()` / `page()` 这些 API

## 5. 配置

```yaml
valkey:
  query:
    enabled: true
    mode: standalone
    username: default
    password: your-password
    nodes:
      - host: 127.0.0.1
        port: 6379
```

如果你本地没开 ACL，也可以去掉 `username`。

## 6. 第一条写入和查询

```java
@Component
public class DemoRunner implements CommandLineRunner {

    private final BookRepository bookRepository;

    public DemoRunner(BookRepository bookRepository) {
        this.bookRepository = bookRepository;
    }

    @Override
    public void run(String... args) {
        Book book = new Book();
        book.setId("book-1");
        book.setTitle("Valkey in Action");
        book.setAuthor("momao");
        book.setPrice(99);

        bookRepository.checkAndCreateIndex();
        bookRepository.save("book-1", book);

        BookQuery q = new BookQuery();

        List<Book> records = bookRepository.queryChain()
            .where(q.title.contains("Valkey"))
            .and(q.author.eq("momao"))
            .list();

        System.out.println(records);
    }
}
```

## 7. 如果这里跑不通，先看哪里

最常见卡点只有 4 个：

1. Valkey 没起，或者地址端口不对
2. Search / JSON 能力不存在
3. 密码不对，`GlideClient` 启动失败
4. APT 没生效，`BookQuery` 没生成
