# 拓扑支持矩阵

这篇文档只回答一个问题：

- 某个能力在 `standalone / read_write_split / cluster` 下到底能不能用

## 1. 总览

| 能力 | standalone | read_write_split | cluster |
| --- | --- | --- | --- |
| 基础查询 `list/page/one/count` | 支持 | 支持 | 支持 |
| `queryChain()` | 支持 | 支持 | 支持 |
| `save / saveAll / saveBatch / deleteAll / updateAll` | 支持 | 支持 | 支持 |
| `updateChain()` | 支持 | 支持 | 支持 |
| `select / projection` | 支持 | 支持 | 支持 |
| `aggregateChain()` 基础子集 | 支持 | 支持 | 仅安全子集 |
| `facetChain()` | 支持 | 支持 | 支持 |
| `distinct / distinctRows / countDistinct` | 支持 | 支持 | 支持 |
| 向量 KNN | 支持 | 支持 | 不支持 |
| 索引治理 `NONE / VALIDATE / RECREATE` | 支持 | 支持 | 支持 |

## 2. cluster 当前安全子集

cluster 下当前明确承诺的聚合子集：

- `GROUPBY`
- `COUNT`
- `SUM`
- `AVG`
- `MIN`
- `MAX`
- `SORTBY`
- `LIMIT`

原因：

- 这些 reducer 可以安全做跨分片合并
- 框架不会为了“看起来透明”返回语义不稳定的结果

## 3. cluster 当前不支持的重点能力

- 向量 KNN
- 不可安全合并的聚合表达式组合

如果业务是 cluster，开发时不要默认假设所有高阶查询都透明支持。

## 4. read_write_split 需要知道的事

- 写请求固定走主节点
- 只读请求按 `read-preference` 选择副本或主节点
- 索引治理命令固定走写路径
- 副本缺索引元数据时，`FT.*` 读请求会回退主库

## 5. 多模块 Spring Boot 项目怎么配

单模块项目：

```java
@EnableValkeyQuery
```

多模块项目：

```java
@EnableValkeyQuery({
    "com.example.account",
    "com.example.order",
    "com.example.search"
})
```

说明：

- `value` 是 `basePackages` 的别名
- 无参时默认回退到启动类所在包
- 多模块项目建议始终显式写包，避免扫描边界不直观
