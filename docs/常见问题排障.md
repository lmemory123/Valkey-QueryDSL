# 常见问题排障

这篇文档只讲用户最容易卡住的错误。

## 1. `创建 GlideClient 失败`

最常见原因：

1. `host / port` 不对
2. `password` 不对
3. Valkey 服务没启动
4. 服务端需要 ACL，但你没传 `username`

优先检查：

```yaml
valkey:
  query:
    username: default
    password: your-password
    nodes:
      - host: 127.0.0.1
        port: 6379
```

如果你是通过环境变量传：

```bash
VALKEY_HOST=127.0.0.1
VALKEY_PORT=6379
VALKEY_USERNAME=default
VALKEY_PASSWORD=your-password
```

## 2. `创建索引失败`

最常见原因：

1. 服务端没有 Search / JSON 能力
2. 注解定义和真实服务端能力不兼容
3. 旧索引 schema 和当前实体定义不一致

优先看：

- 当前索引治理模式是不是 `VALIDATE`
- 你是不是在旧索引上直接换了字段类型

补充说明：

- `TAG` 字段当前不会生成 `SORTABLE`
- `TEXT` 当前不会默认追加 `WITHSUFFIXTRIE`

## 3. 写入后立刻查不到

这通常不是写入失败，而是索引还没收敛。

表现：

- `save(...)` 已成功
- 紧接着 `count()` / `list()` 暂时看不到

处理方式：

- 对业务侧接受最终一致性
- 对测试侧显式等待索引可见

最小等待示例：

```java
long deadline = System.currentTimeMillis() + 10_000;
while (System.currentTimeMillis() < deadline) {
    if (bookRepository.count(new BookQuery().author.eq("momao")) > 0) {
        break;
    }
    Thread.sleep(200);
}
```

如果你是在测试里等索引，建议：

- 每次轮询都允许短暂失败重试
- 不要把第一次 `count()` 失败直接当成功能错误

## 4. cluster 下某些能力不能用

这不是 bug，而是当前框架明确收口的边界。

目前 cluster 明确不支持或不完全支持的重点能力：

- 向量 KNN
- 不可安全合并的聚合表达式组合

## 5. `BookQuery` / `SkuQuery` 没生成

最常见原因：

1. 没加 APT
2. IDE 没启用 annotation processing
3. 用了错误版本的 `valkey-query-processor`

Maven 最小配置：

```xml
<annotationProcessorPaths>
    <path>
        <groupId>io.github.lmemory123</groupId>
        <artifactId>valkey-query-processor</artifactId>
        <version>1.1.0-RC5</version>
    </path>
</annotationProcessorPaths>
```

## 6. 多模块项目扫不到 Repository

优先检查 `@EnableValkeyQuery`。

单模块推荐：

```java
@EnableValkeyQuery
```

多模块推荐显式写包：

```java
@EnableValkeyQuery({
    "com.example.user",
    "com.example.order"
})
```

## 7. 普通 push 为什么没有打包发布

现在仓库已经明确收口成：

- `push main`：只跑 CI 测试
- 手工执行 `release-publish.sh`：才发 Maven Central
- `push v* tag`：才生成 GitHub Release
