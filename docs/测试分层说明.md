## 建议执行顺序

开发时：

1. 先跑纯回归
2. 再跑真实 standalone
3. 涉及拓扑改动时，再跑 read_write_split / cluster

合并前：

1. standalone 真实测试必须通过
2. 涉及拓扑或路由改动时，主从和 cluster 真实测试必须通过
